# 먹어볼래? | 블로그 광고 필터링 및 리뷰 요약 서비스

> 광고성 블로그 리뷰를 판별 및 실제 방문자 리뷰를 요약·해시태그화하여 더 신뢰도 높은 맛집 탐색을 돕는 AI 기반 서비스입니다.

## 프로젝트 소개

기존 맛집 탐색 과정에서는 광고성 블로그 글이 많아 정보 신뢰도가 떨어지고 실제 방문자 리뷰도 양이 많아 핵심 내용을 빠르게 파악하기 어렵다는 문제가 있습니다.  
이 프로젝트는 이러한 불편을 해결하기 위해 **광고성 블로그 필터링**, **리뷰 요약 결과 제공**, **감성 해시태그 추출** 기능을 결합한 서비스로 구현했습니다.

- **개발 기간**: [프로젝트 기간 입력]
- **개발 인원**: 3명
- **담당 역할**: 개념설계 및 상세설계, 광고/비광고 블로그 데이터 수집·전처리 및 KoBERT 학습 실험 참여, FastAPI 기반 중앙 파이프라인 구현, 리뷰 요약 결과 JSON 후처리, Mecab+KNU 감성사전 기반 종합 리뷰 점수 및 긍·부정 해시태그 생성
- **형태**: AI + 웹 서비스 통합 프로젝트

## 문제 정의

- 광고성 블로그 리뷰가 많아 식당 정보의 신뢰도가 낮음
- 방문자 리뷰 수가 많아 핵심 내용을 빠르게 확인하기 어려움
- 평점만으로는 사용자의 방문 목적에 맞는 식당 선택이 어려움

이 프로젝트는 위 문제를 해결하기 위해  
**광고성 콘텐츠를 필터링**
**실제 리뷰 요약 결과를 제공**
**감성 키워드를 함께 제공**으로 사용자 탐색 비용을 줄이는 데 초점을 맞췄습니다.

---

## 주요 기능

- **광고성 블로그 판별**
  - KoBERT 기반 분류 모델을 활용해 광고/비광고 블로그를 구분
  - 게시글별 광고 여부를 판별해 리뷰 신뢰도 판단에 활용

- **리뷰 요약 결과 제공**
  - 리뷰 요약 시스템이 생성한 긍정/부정/전체 요약 결과 제공
  - 많은 방문자 리뷰의 핵심 내용을 빠르게 확인 가능

- **감성 해시태그 추출**
  - 감성 사전을 기반으로 긍정/부정 키워드 및 특징 키워드 추출
  - 맛, 서비스, 분위기, 가격, 재방문 의사 등 핵심 요소를 시각화

- **검색 결과 및 상세페이지 제공**
  - 식당 검색 결과, 리뷰 요약, 감성 키워드, 상세 정보 제공
  - 카카오 지도 API를 활용해 위치 정보까지 함께 확인 가능

## 담당 구현

- 서비스 전체 구조에 대한 **개념설계 및 상세설계** 수행
- 사용자 검색 → 데이터 수집 → 광고 판별 → 리뷰 요약 결과 통합 → 감성 분석 → 화면 출력으로 이어지는 전체 서비스 흐름 설계
- **네이버 검색 API 및 블로그 본문 크롤링 기반 광고/비광고 데이터 수집 및 전처리 수행**
- 광고성 키워드 기준 라벨링, 텍스트 정제, 토큰화 과정을 거쳐 **KoBERT 학습용 데이터셋 구성**
- **KoBERT 광고/비광고 분류 모델 학습 과정 참여**
- 입력 토큰 길이 변경 및 데이터 구성 조정을 통해 성능 개선 실험 참여
- **FastAPI 기반 중앙 파이프라인 구현**
- 광고/비광고 판별 결과, 리뷰 요약 결과, 감성 분석 결과를 통합해 상세페이지용 JSON 응답 생성
- **리뷰 요약 시스템이 반환한 긍정/부정/전체 요약 JSON 후처리 로직 구현**
- **Mecab 형태소 분석과 KNU 감성사전 매칭을 통해 종합 리뷰 점수, 긍정 해시태그, 부정 해시태그를 생성하는 파이프라인 구현**
- 카카오 지도 API와 연동되는 상세페이지 데이터 구조를 설계하고, 프론트엔드에서 바로 사용할 수 있는 형태로 응답 가공

## 설계 및 구현 포인트

### 1. AI 결과를 실제 서비스 흐름으로 통합
검색 결과, 광고/비광고 판별, 리뷰 요약, 감성 분석 결과를 각각 분리하지 않고  
FastAPI 백엔드에서 하나의 상세페이지 응답으로 통합해 서비스에 연결했습니다.

### 2. 광고/비광고 분류용 데이터셋 구축
네이버 블로그 본문을 크롤링하고 텍스트 전처리 및 라벨링을 수행해  
KoBERT 분류 모델 학습에 활용할 수 있는 데이터셋을 구성했습니다.

### 3. FastAPI 기반 중앙 파이프라인 구현
식당 정보, 광고 판별 결과, 리뷰 요약 결과, 감성 분석 결과를 하나로 묶어  
상세페이지에서 사용할 수 있는 JSON 구조로 통합했습니다.

### 4. 리뷰 요약 결과 후처리
리뷰 요약 시스템이 생성한 JSON 결과를 그대로 노출하지 않고  
Mecab 형태소 분석과 KNU 감성사전 매칭을 거쳐 종합 리뷰 점수와 긍·부정 해시태그로 변환했습니다.

### 5. 백엔드 중심의 응답 구조 설계
광고 확률, 리뷰 요약, 감성 해시태그, 지도 정보가 한 화면에서 자연스럽게 보이도록  
프론트엔드에서 바로 사용할 수 있는 JSON 응답 구조를 설계했습니다.

## 트러블슈팅

### 광고/비광고 데이터 전처리
블로그 원문에는 HTML 태그, 특수문자, 중복 표현, 광고성 문구가 혼재되어 있어 바로 학습 데이터로 사용하기 어려웠습니다.  
이를 해결하기 위해 텍스트 정제, 토큰화, 라벨링 기준을 적용해 KoBERT 학습에 적합한 형태로 데이터를 가공했습니다.

### KoBERT 입력 길이 조정
초기 학습에서는 입력 길이 제한으로 문맥 손실이 발생했습니다.  
이를 개선하기 위해 128 토큰에서 512 토큰으로 입력 길이를 확장해 문맥 보존 성능을 높이는 방향으로 실험했습니다.

### 리뷰 요약 결과 후처리
리뷰 요약 결과를 그대로 사용자에게 보여주면 핵심 정보가 직관적으로 드러나지 않는 문제가 있었습니다.  
이를 해결하기 위해 Mecab 형태소 분석과 KNU 감성사전 매칭을 적용해 종합 리뷰 점수와 긍정/부정 해시태그를 생성하는 후처리 로직을 구현했습니다.

### 감성사전 매칭 정확도 개선
리뷰 표현이 다양해 감성 키워드를 일관되게 분류하는 데 어려움이 있었습니다.  
이를 해결하기 위해 형용사 표현을 명사화하고 음식점 리뷰에서 자주 등장하는 요소를 기준으로 감성사전 범주를 정리해 매칭 정확도를 높였습니다.
